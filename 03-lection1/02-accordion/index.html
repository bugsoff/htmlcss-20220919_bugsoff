Flexbox vs. CSS Grid (1)

When I first started programming, I only knew how to use flexbox. So naturally, I only used flexbox. By no means did I think that flexbox was any better or any worse than CSS grid, I just never learned how to use it properly.

Once I started to get more responsibilities at my job, I realized that maybe flexbox wasn’t always the go-to answer for every project. It was getting in the way with the different layouts that I was building and it was causing more trouble than it was worth.

In this article, I’m going to lay out the differences between the flexbox and CSS grid. Then, I will discuss when the best use case for each is.

Flexbox vs. CSS Grid (2)

Flexbox

Flexbox is one-dimensional, meaning that you can really only design your page in one direction or the other. It’s not designed to support both rows and columns at the same time.

The main properties that I typically use with Flexbox are display, flex-direction,align-items, and justify-content. I suggest checking out the MDN Web Docs for more information on all of the other properties that Flexbox supports.

Flexbox vs. CSS Grid (3)

Grid

CSS Grid is two-dimensional, meaning that it’s great for diving up your webpage into several different regions. As long as you know what you are doing and what you want, you have full control of how to set up your webpage.
